# Security Policy

## Supported Versions

Security fixes target the latest tagged module release and current `main`.

## Reporting a Vulnerability

Please do not open a public issue for secrets, credential exposure, or exploitable vulnerabilities.

Report privately to the maintainers through an agreed private channel. Include:

- affected module version or commit
- impact and attack path
- reproduction steps
- any plans, logs, or screenshots with secrets redacted

We will acknowledge receipt, assess severity, and coordinate a fix before public disclosure.
